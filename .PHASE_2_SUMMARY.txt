🤖 AGENTES MULTI-AGENTE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 👨‍💼 MANAGER (Director Ejecutivo)
   ├─ Toma requerimientos del usuario
   ├─ Crea plan detallado de ejecución
   ├─ Dirige la "Daily" (reunión de equipo)
   └─ Publica: "iniciando", "planificando", "completada"

2. 👨‍💻 ESPECIALISTA (Ejecutor)
   ├─ Ejecuta tareas específicas
   ├─ Investigación, redacción, análisis
   ├─ Trabaja bajo dirección del Manager
   └─ Publica: "trabajando", "analizando", "documentando"

3. ✅ QA/REVIEWER (Revisor de Calidad)
   ├─ Revisa exhaustivamente el trabajo
   ├─ Busca errores e inconsistencias
   ├─ Valida contra requerimientos originales
   └─ Publica: "revisando", "validando", "aprobada/rechazada"

🎯 FLUJO COMPLETO DE UNA TAREA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Cliente HTTP
   └─ POST /api/task {"title": "...", "description": "..."}

2. FastAPI
   ├─ Genera task_id único
   ├─ Publica evento: "Manager recibida"
   └─ Devuelve: {task_id, status: "pending"}

3. Background Task (execute_crew_task)
   ├─ Crea instancia AgencyTeam
   └─ Llama: agency_team.execute_task()

4. Crew de Agentes
   ├─ TAREA 1: Manager
   │  ├─ Analiza requerimiento
   │  ├─ Crea plan detallado
   │  └─ Publica eventos → Redis
   │
   ├─ TAREA 2: Especialista (contexto de Manager)
   │  ├─ Ejecuta plan
   │  ├─ Investiga, redacta, analiza
   │  └─ Publica eventos → Redis
   │
   └─ TAREA 3: QA (contexto de Especialista)
      ├─ Revisa exhaustivamente
      ├─ Valida requerimientos
      └─ Publica eventos → Redis

5. Redis Channel: "agency_events"
   └─ Broadcast a Gateway WebSocket

6. Gateway WebSocket
   └─ Broadcast a todos los clientes conectados

7. Frontend
   └─ Renderiza chat en tiempo real
